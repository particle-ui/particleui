# Design Brief — ParticleUI

_Generated by /design-grill on 2026-05-16. Update this file when design decisions change._

## Product Context
- **Category:** Component library — documentation site + marketing site. Both surfaces carry equal weight and must feel like one coherent product.
- **Target user:** Design-aware developers who know what good UI looks like and are willing to pay for components that match that standard. They've seen Linear/Vercel quality and want it for their own products.
- **Primary device/context:** Desktop-primary (developers at workstations). Mobile must work but is secondary.
- **Defining screen:** The landing page / homepage — first impression, brand case. If this doesn't stop the scroll, nothing else matters.

## Brand Personality
**5 adjectives:** precise, confident, dark, craft-forward, understated
**Emotional goal:** Capable — "I can build things that look this good." The gap between a developer's idea and production-quality output just closed.
**Tone:** A tool made by designers who code. Knows exactly what it is, doesn't shout about it. The quality speaks. No flash without function, no decoration without purpose.

## Visual References
- **Linear** — tight compressed typography, dark surfaces, product confidence. Borrow: heading scale, whitespace discipline, dark neutral palette.
- **Vercel (marketing site)** — bold headings, generous whitespace, electric accent moments. Borrow: marketing boldness, hero drama, how accent color creates hierarchy without color overload.
- **shadcn/ui (docs)** — code-first hierarchy, scannable docs, no visual noise. Borrow: docs clarity, information density decisions, how code blocks are treated as first-class UI.

## What to Avoid
- **Chakra UI / MUI** — generic blue primary, enterprise-gray surfaces, rounded cards that feel like a template. Never the "component library blue" look.
- **Overdesigned docs** — animation or decoration that gets in the way of scanning. Expressive must remain disciplined.

## Color System
- **Default mode:** Dark-first. Light mode supported via theme system (Mono, Rose, Slate themes exist), but not the primary experience. Add a proper light default later.
- **Temperature:** Warm — espresso surfaces with amber hue whisper, cream accent.
- **Primary color direction:** Cream/warm-white accent on near-black espresso. Not a colored primary — the accent is intentionally low-chroma.
- **Palette approach:** Single accent + warm neutrals. No competing accent colors.
- **Saturation:** Muted — the accent is almost achromatic (`oklch(96% 0.01 80)`). Semantic colors (success/error/warning) provide the only vivid color moments.
- **Established token values:**
  - Background: `oklch(4% 0.005 60)` — warm near-black (espresso)
  - Surface-1: `oklch(8% 0.004 60)` — warm dark card
  - Accent: `oklch(96% 0.01 80)` — cream (heavy cream, not white)
  - Text-1: `oklch(97% 0.005 80)` — warm white headings
  - Text-2: `oklch(72% 0.003 80)` — warm light gray body
  - Text-3: `oklch(50% 0.002 80)` — secondary / captions (minimum for most text)

## Typography
- **Typeface:** Geist (sans) + Geist Mono — already established. Geist is neutral-technical with Vercel confidence. Geist Mono for all code.
- **Heading style:** Mixed by surface:
  - **Marketing site:** Compressed — tight tracking (`tracking-[-0.04em]`), heavy weight (`font-bold` to `font-extrabold`), dramatic scale. Headings are brand statements.
  - **Docs site:** Airy — normal tracking, medium weight (`font-semibold`), calm scale. Headings are navigation aids, not statements.
- **Body:** 15px minimum everywhere. Leading 1.7–1.75. Weight 400–500.
- **Scale approach:** Balanced — not pure editorial, not pure dense. Marketing tips toward editorial, docs tips toward dense.

## Component Style
- **Corner radius:** 12px (`rounded-xl`) — friendly-precise. Not developer-sharp, not consumer-pill. Approachable without being soft.
- **Elevation:** Flat — borders only, no drop shadows on cards. Surface hierarchy via lightness steps, not shadow.
- **Density:** Moderate — balanced between marketing spaciousness and dashboard density.
- **Borders:** Light (`oklch(100% 0 0 / 0.07)`) — present but not heavy. Hover state at 0.14 opacity.
- **Overall:** Rounded, flat, balanced, light borders — premium developer tool. Craft-forward without the developer-utilitarian register.

## Animation
- **Level:** Expressive throughout — but disciplined. Every animation earns its place. Motion is precise and purposeful, never decorative for its own sake.
- **Distinction:** The product UI animates expressively. Showcased components animate to demo their own capabilities. These are two separate layers — the UI chrome never competes with the showcased particle effects.
- **Key interactions to make feel great:** Page transitions, component preview reveals in the gallery, the theme studio token updates.
- **Easing:** `ease-out` for entrances, `ease-in-out` for state changes. No bouncing or spring unless it's a particle effect being showcased. Duration: 150–300ms for UI chrome, longer allowed for showcase moments.
- **Constraint:** Docs animations must not interfere with reading/scanning. Expressive in docs = scroll reveals and hover states, not looping or distracting motion.

## Dark Mode
- Dark-first, light mode later. Token system supports both. Don't block design decisions on light mode parity — design dark-first, add light as a secondary concern.

## Decisions Made
| Decision | What was decided | Why |
|----------|-----------------|-----|
| Primary surface | Both marketing + docs equally define the brand | One coherent product, not two separate visual systems |
| Target user | Design-aware developers (all experience levels) | The $149 Pro tier, MCP server, and OKLCH docs all signal this user |
| Emotional goal | Capable | Empowerment: the gap between idea and production-quality output |
| Accent color | Cream, not a colored primary | Understated confidence — color restraint IS the brand statement |
| Light mode | Dark-first, light later | Product identity is dark; light mode is a theme option, not the default |
| Heading scale | Compressed (marketing) + Airy (docs) | Different jobs: brand statement vs navigation aid |
| Animation | Expressive but disciplined | The product is its own best demo — but never noisy |
| Corner radius | 12px (keep current) | Right balance of approachable and precise |

## Ruled Out
_Directions explicitly rejected during this session. Do not re-propose without reopening the decision._

| Direction | Why it was rejected |
|-----------|-------------------|
| Generic blue primary | The "component library blue" look — Chakra/MUI register. ParticleUI must not look like an enterprise component library. |
| Visual noise / undisciplined animation | "Expressive" was chosen but must remain disciplined. Looping or purposeless motion violates the "understated" personality. |
| Brochure-ware docs | Docs should feel like a premium developer tool, not a marketing site landing page. |
